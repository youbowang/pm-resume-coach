# 产品经理简历表达规则

## 好的简历 bullet 结构

建议使用以下结构：

```text
场景 / 目标 + 行动方法 + 产出结果 + 能力关键词
```

示例：

```text
围绕校园二手交易场景，访谈学生用户并归纳交易信任、商品筛选、沟通效率等核心需求，输出需求优先级和核心交易流程。
```

## 弱表达识别

以下表达通常偏弱：

- “参与项目”
- “协助完成”
- “负责整理”
- “学习了”
- “体验了”
- “输出了文档”

这些词不是不能用，但需要补充个人贡献、方法、产出和影响。

## 不应过度包装

不能无依据把：

- “参与”改成“主导”
- “协助”改成“独立负责”
- “整理资料”改成“制定战略”
- “做了体验”改成“完成系统性评测”
- 没有数据的项目写成“提升 30% 转化率”

如果原始材料没有证据，应提示用户补充真实信息。

## 产品经理实习简历关键词

常见关键词包括：

- 用户研究
- 需求分析
- 竞品分析
- 原型设计
- 数据分析
- 项目推进
- 跨职能协作
- Prompt 设计
- AI 效果评测
- 知识库 / RAG
- 真实性风险控制
